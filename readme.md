# Serverless AI ATS (HROps)

A serverless Applicant Tracking System (ATS) that automatically evaluates candidate resumes (PDF or DOCX) against job descriptions using Amazon Bedrock and AWS Lambda.

## Architecture

The system uses an event-driven architecture based on managed AWS services:

1. **Frontend:** Candidates upload CVs via a presigned S3 URL generated by API Gateway. The active job posts (`jobs.json`) are fetched from a static S3 bucket.
2. **Storage:** The uploaded resume is stored in an Amazon S3 bucket.
3. **Processing:** S3 Put object events trigger an AWS Lambda function. The function extracts text from the resume using `pypdf` and `python-docx`.
4. **AI Evaluation:** Lambda sends the extracted text and the target job description to Amazon Bedrock (`amazon.nova-pro-v1:0`) for a structured evaluation.
5. **Database:** The candidate's score, missing skills, and detailed feedback are saved to Amazon DynamoDB.
6. **Notifications:** If the candidate passes the minimum match score, an Amazon SNS email notification is sent to the HR team.

## Prerequisites

* AWS CLI and AWS SAM CLI installed (or use AWS Cloud9).
* Python 3.11 or newer.
* AWS account with model access enabled for **Amazon Nova Pro** (`amazon.nova-pro-v1:0`) in Amazon Bedrock.

## Deployment

1. Build the serverless application:

   ```bash
   sam build
   ```

2. Deploy the infrastructure to AWS:

   ```bash
   sam deploy --guided
   ```

   During the guided deployment, provide necessary parameters like `MinMatchScore` (e.g., 75) and `JOBS_BUCKET_NAME` (the S3 bucket where `jobs.json` will be stored).

## Configuration & Usage

* **Single Source of Truth (`jobs.json`)**: Upload your `jobs.json` file to a public or static S3 bucket.
* **Frontend Setup**: Update the `JOBS_JSON_URL` variable in `frontend/index.html` to point to the public S3 URL of your `jobs.json` file.
* Open `frontend/index.html` in a web browser, select a job position, and upload a candidate CV.

## Test Data

You can locally generate realistic test resumes (PDF/DOCX formats) using a local LLM environment (Ollama) to test the ATS evaluation pipeline.

```bash
pip install -r requirements.txt
python generate_test_data.py
```
